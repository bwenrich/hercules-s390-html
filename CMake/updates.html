Using CMake to Build Hercules Version 4.0

What's new in CMake for Hercules

Winter 2018 Update

Windows Builds are Supported

CMake can be used to build Hercules on Windows systems. One may use the Microsoft Visual Studio Integrated Development Environment to build, or you can use MSBuild, NMake, or JOM from the command line. The required external packages BZip2, PCRE, and Zlib are built when Hercules is built, just as the current 1Stop.cmd builds SoftFloat-3a. See Building Hercules for Windows for complete details.

Note: Windows build support changes how three environment variables are used when building Hercules. These are bzlib_dir, pcre_dir, and zlib_dir. See External Package Incompatibilities (BZip2, PCRE, Zlib) and Environment Variable Incompatibilities on the Building Hercules for Windows page for details and actions to take.


CMake Can Build BZip2, Zlib, and for Windows, PCRE if needed

If BZip2 and/or Zlib are not installed on your UNIX-like system, or the installed versions are older releases than the versions of those packages available in the Hercules-390 github project, the CMake build will clone and build missing or downlevel packages from the Hercules-390 github project.

For builds on Windows systems, Bzip2, PCRE, and Zlib are built from the Hercules-390 project.

CMake command line options are available to force the use of the system-installed packages (UNIX-like systems only), to use previously built versions, or to exclude the packages and their capabilities from the Hercules build. See the descriptions of the BZIP2 and BZIP2_DIR options and the corresponding options for PCRE and ZLIB in Standard CMake Build Options
